# Compression Replayability Matrix

This matrix compares continuity dimensions against compression mechanisms and the current public evidence set. Assessments are conservative and may change after additional evidence collection.

| Continuity Dimension | Compression Mechanism | Public Evidence | Replayability Impact | Current Assessment | Claim Level | Evidence Refs |
| --- | --- | --- | --- | --- | --- | --- |
| transcript continuity | Compaction summary, condensed view, successor transcript, kept head/tail context | OpenClaw compaction docs; OpenHands condenser docs | Replay can preserve visible summary and recent context while reducing direct access to older raw turns unless original ranges remain linked. | Public evidence suggests transcript continuity may survive at coarse granularity while fine-grained transcript replay depends on retained source ranges and successor metadata. | inferred | openclaw-doc-compaction; openhands-doc-condenser-architecture |
| causal lineage continuity | Summary generation, forgotten event filtering, prompt-view reconstruction | OpenHands condenser docs; OpenHands events docs; OpenClaw trajectory docs | Replay may show what state was supplied after compression without preserving every causal step that produced that state. | Public evidence indicates lineage surfaces exist, but causal sufficiency after compression remains unresolved. | unresolved | openhands-doc-condenser-architecture; openhands-doc-events-architecture; openclaw-doc-trajectory-bundles |
| tool/action continuity | Tool-call pairing preservation, event logs, action/observation flows | OpenClaw compaction docs; OpenHands events docs | Replay fidelity improves when call/result pairs and typed action events survive compression boundaries. | Public evidence suggests tool/action continuity can be partly protected, but exact tool-result survivability remains an evidence question. | inferred | openclaw-doc-compaction; openhands-doc-events-architecture |
| memory continuity | Memory flush, memory files, recall/search surfaces | OpenClaw memory docs; OpenClaw compaction docs | Replay after compression may depend on whether memory writes and recall results are captured with provenance. | Public evidence suggests memory can preserve selected continuity outside active context, but memory replay adequacy is unresolved. | inferred | openclaw-doc-memory-overview; openclaw-doc-compaction |
| environment continuity | Runtime/workspace surfaces outside compressed text | OpenHands agent server docs; OpenClaw trajectory docs | Text compression does not by itself preserve filesystem, container, browser, dependency, or external-service state. | Public evidence makes environment continuity relevant, but current sources do not establish complete reconstruction. | unresolved | openhands-doc-agent-server-architecture; openclaw-doc-trajectory-bundles |
| temporal ordering continuity | Condensation event insertion, successor transcript checkpointing, append-only event logs | OpenHands condenser/conversation docs; OpenClaw compaction docs | Summaries may preserve sequence at coarse granularity while reducing fine-grained replay fidelity. | Public evidence suggests ordering summaries may preserve broad sequence while omitted ranges need event IDs and timestamps for audit-grade replay. | inferred | openhands-doc-condenser-architecture; openhands-doc-conversation-architecture; openclaw-doc-compaction |
| contradiction continuity | LLM or heuristic summarization of disputed context | Compression docs show summary mechanisms, but current evidence does not show contradiction-specific retention guarantees. | Replay can lose the difference between conflict, uncertainty, and resolved conclusion if contradiction refs are not preserved. | Contradiction survivability is a hypothesis requiring targeted evidence. | hypothesis | openclaw-doc-compaction; openhands-doc-condenser-architecture |
| replayability continuity | Visible replay artifacts plus compression metadata | OpenClaw trajectory docs; OpenHands browser recording, conversation, event, and condenser docs; rrweb docs | Replay remains operationally meaningful only when compressed context can be linked to omitted ranges, source hashes, and state refs. | Visible replay and compressed context surfaces are observed, but post-compression causal replay adequacy is unresolved. | unresolved | openclaw-doc-trajectory-bundles; openhands-doc-browser-session-recording; openhands-doc-condenser-architecture; shared-rrweb-readme |
| recovery-loop continuity | Auto-compaction on overflow, manual condensation request, fallback during summarization | OpenClaw compaction docs; OpenHands condenser docs | Recovery replay may lose why continuation succeeded or failed if errors, retries, fallback choices, and compaction boundaries are omitted. | Public evidence suggests recovery events can trigger compression, but recovery path reconstruction remains partial. | inferred | openclaw-doc-compaction; openhands-doc-condenser-architecture |
| intent continuity | Summary text, guided compaction instructions, action-sensitive memory boundaries | OpenClaw compaction and memory docs; OpenHands condenser docs | Replay can drift if task intent survives as a short summary but constraints, authority, expiry, or scope changes are omitted. | Intent survivability drift is a plausible compression risk, not a validated target-system finding. | hypothesis | openclaw-doc-memory-overview; openclaw-doc-compaction; openhands-doc-condenser-architecture |
