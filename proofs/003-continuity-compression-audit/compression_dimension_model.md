# Compression Dimension Model

The dimensions below define continuity properties that may be affected by compression. Each row is preliminary and subject to evidence validation.

Allowed claim levels: `observed`, `inferred`, `hypothesis`, `unresolved`.

| Dimension | Definition | Why Compression Affects It | Observable Public Evidence | Unresolved Questions | Current Claim Level |
| --- | --- | --- | --- | --- | --- |
| transcript continuity | Preservation of the visible conversation transcript, including user turns, assistant turns, summaries, and successor transcript boundaries. | Compression may replace detailed prior turns with a compact summary or create a new active transcript, changing what remains visible to later model turns. | OpenClaw compaction docs describe summaries, full history on disk, recent-turn preservation, and successor transcripts. OpenHands condenser docs describe condensed views and event summaries. | Are all original transcript segments retained with stable references after compression? Are omitted ranges visible to auditors? | observed |
| causal lineage continuity | Preservation of the causal links between prior context, decisions, tool results, memory recalls, and later actions. | Summaries can preserve conclusions while omitting intermediate reasons, failed branches, rejected evidence, or source-specific cause paths. | OpenHands condenser docs describe `forgotten_event_ids`; OpenClaw trajectory docs describe replay artifacts; OpenHands event docs describe typed event flows. | Do public artifacts preserve enough prompt-building, memory, tool-result, and environment references to reconstruct why a later action occurred? | unresolved |
| tool/action continuity | Preservation of tool calls, observations, tool results, and call/result pairing across compression boundaries. | Compression can summarize action sequences or omit detailed results, reducing the ability to replay exact operational steps. | OpenClaw compaction docs describe preserving assistant tool calls with matching `toolResult` entries when splitting history. OpenHands event docs describe action and observation event flows. | Are large or redacted tool results preserved by hash, pointer, or content? Are tool schema versions and execution settings retained? | inferred |
| memory continuity | Preservation of durable memory, working notes, recall queries, recall results, and memory flush boundaries. | Compression may rely on memory flush or recall surfaces to move continuity out of the active context; missing memory lineage can change later reconstruction. | OpenClaw memory docs describe Markdown memory files, memory search/get surfaces, truncation status, and automatic memory flush before compaction. | Are memory writes, recalls, indexes, truncation boundaries, and promotions captured with enough provenance for replay? | inferred |
| environment continuity | Preservation of workspace, filesystem, runtime, sandbox, browser, container, and external-service state references. | Summaries usually compress text history, not the external state that tool calls observed or modified. | OpenHands agent server docs describe workspace and remote execution surfaces. OpenClaw trajectory docs are relevant to runtime metadata and replay artifacts. | What environment hashes, filesystem diffs, dependency versions, container IDs, and external state references survive compression? | unresolved |
| temporal ordering continuity | Preservation of event order, timestamps, compaction timing, retry timing, and before/after relationships. | Summary text can collapse many ordered events into unordered prose, and condensation can introduce new events that stand in for older ranges. | OpenHands conversation/event docs describe append-only events and state updates. OpenHands condenser docs describe condensation events with insertion offsets. OpenClaw compaction docs describe auto/manual compaction and successor transcripts. | Are exact timestamps, event IDs, and compaction boundaries retained for every omitted range? | inferred |
| contradiction continuity | Preservation of conflicting evidence, reversals, stale assumptions, or corrected beliefs across compression. | Summaries can flatten contradiction into a single resolved statement, making later auditors unable to distinguish disputed from settled context. | Public evidence shows summary generation and condensation mechanisms, but current sources do not establish contradiction-preservation rules. | Do summaries preserve conflicts, source authority, freshness, and rejected alternatives? Are contradiction refs captured separately? | hypothesis |
| replayability continuity | Preservation of enough post-compression lineage to make replay operationally meaningful. | Replay may show a final summary and later actions while losing the detailed state that made the summary adequate or inadequate. | OpenClaw trajectory docs, OpenHands browser recording docs, OpenHands conversation/event docs, and rrweb docs provide visible replay or event surfaces. | What minimum bundle is sufficient after compression: original ranges, summary, hashes, omitted IDs, tool results, memory refs, and environment refs? | unresolved |
| recovery-loop continuity | Preservation of retries, fallback models, compaction-on-overflow, stuck detection, aborts, and resumptions. | Compression can occur as part of recovery from context overflow; if recovery events are summarized away, causal reconstruction loses the path from error to continuation. | OpenClaw compaction docs describe compaction and retry on context overflow and fallback behavior for compaction attempts. OpenHands condenser docs describe manual condensation after context errors. | Are recovery attempts, fallback choices, errors, and cleanup actions preserved with source refs and timestamps? | inferred |
| intent continuity | Preservation of user intent, task constraints, changed objectives, and action boundaries across compressed context. | Summaries may carry a simplified objective while losing authority, scope, expiry, or safety constraints that were explicit earlier. | OpenClaw memory docs discuss action-sensitive memories and boundaries. Compression docs describe guided manual compaction. OpenHands condenser docs describe summary generation for reasoning. | Do summaries retain source authority, timing, approvals, prohibitions, and changed intent without drift? | hypothesis |
