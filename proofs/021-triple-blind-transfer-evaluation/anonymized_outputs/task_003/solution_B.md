# Solution B

Refresh the SLA export and compare it with frontline ticket ages before reassigning staff. If the enterprise queue is still aging, move trained agents first and avoid moving new hires who are not ready for those cases. The VIP list should be updated before prioritization is finalized.

The manager can request reassignment, but workforce training limits should control which agents move. Product's fix is useful context, but support should verify whether ticket volume actually fell.

Use a one-hour checkpoint. If enterprise age improves without damaging onboarding coverage, keep the temporary assignment. If onboarding begins aging or the moved agents cannot handle enterprise cases, revert and escalate for overtime or specialist backup.
