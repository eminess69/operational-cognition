# Solution A

Refresh the scanner data before making a final decision because the current baseline is too old for a Friday risk call. In parallel, ask product to reproduce the partner break and security to state whether the exploit signal applies to the affected component.

If the refreshed scan shows an active issue and the exploit signal is relevant, block the release or require the patch with partner mitigation. If the scan is clean or the signal is unrelated, request a short exception extension with compensating controls and audit signoff.

The partner contract matters for prioritization, but it should not erase the security deadline. The decision record should name who accepted risk, which evidence was current, and when the exception must be revisited.
