# Contradiction Dimension Model

The dimensions below define what a contradiction survivability audit may need to recover from public artifacts. Each row is preliminary and subject to evidence validation.

Allowed claim levels: `observed`, `inferred`, `hypothesis`, `unresolved`.

| Dimension | Definition | Why It Matters | Public Evidence Surfaces | Flattening Risks | Unresolved Questions | Current Claim Level |
| --- | --- | --- | --- | --- | --- | --- |
| contradiction retention | Preserving explicit conflicts between instructions, observations, memory, summaries, tool results, or operational assumptions. | A later explanation can be wrong if it treats a contested premise as settled. | OpenClaw compaction and trajectory documentation; OpenHands condenser, event, and conversation documentation. | Summaries can select one side of a conflict, omit the losing side, or phrase conflict as resolution. | Are contradictions retained with stable refs to original evidence and later resolution state? | unresolved |
| uncertainty retention | Preserving uncertainty markers, confidence limits, unknowns, caveats, and incomplete-evidence boundaries. | Reconstruction needs to know what was known, unknown, inferred, or merely assumed at decision time. | Summary, condensation, transcript, event, and replay surfaces that expose omitted ranges or caveats. | Compression can convert uncertain statements into declarative prose or drop caveats as low-salience detail. | Which uncertainty markers survive compaction, memory recall, replay views, and restart flows? | unresolved |
| stale assumption survivability | Preserving assumptions that were once active but later became outdated, contradicted, or expired. | Long-horizon failures can arise when a system continues to act on outdated context. | Memory, compaction, event timelines, recovery logs, and summaries with temporal boundaries. | Stale assumptions can remain in memory or summary text without validity windows or supersession refs. | Can public artifacts distinguish current facts from stale assumptions retained for context? | hypothesis |
| rejected-alternative survivability | Preserving alternatives considered and rejected, including why they were rejected and what evidence supported rejection. | Reconstructing reasoning pressure often requires knowing which paths were discarded. | Event logs, trajectory bundles, transcript ranges, summary artifacts, and issue records when public. | Summaries can keep only the chosen path, making replay appear more deterministic than the original context. | Are rejected alternatives referenced, omitted, or unrecoverable after condensation and replay? | unresolved |
| source-authority survivability | Preserving who or what asserted a claim, whether authority changed, and whether later summaries preserve that authority. | Contradictions can depend on source priority, user authorization, tool output, documentation, or inferred state. | Transcript roles, event schemas, memory writes, recall outputs, tool observations, and summary provenance. | Authority can collapse when user instructions, assistant inferences, tool output, and memory all become undifferentiated summary text. | Which artifacts preserve source authority and precedence after compression or memory promotion? | unresolved |
| temporal contradiction survivability | Preserving conflicts that only make sense across time, including before/after states, corrections, supersessions, retries, and restarts. | A contradiction may be valid at one time and stale at another; replay must recover the timeline. | Event timestamps, conversation state, compaction boundaries, trajectory timelines, and recovery/restart records. | Summaries can collapse ordered conflicts into timeless prose or omit the correction boundary. | Are temporal validity windows and supersession points retained with event IDs or hashes? | inferred |
| replay ambiguity survivability | Preserving ambiguity when replay surfaces cannot uniquely explain why a later action happened. | Visible replay can support review while still leaving causal or evidentiary ambiguity unresolved. | Browser/session replay, trajectory bundles, event logs, condensed views, and public manifests. | Replay views can present a clean path without showing omitted alternatives, uncertain state, or missing provenance. | Which replay surfaces preserve ambiguity markers instead of implying a complete causal path? | unresolved |
| memory contradiction survivability | Preserving conflicts between memory writes, recall results, summaries, and current context. | Memory can outlive the context that created it and can reintroduce stale or conflicting premises later. | OpenClaw memory and compaction documentation; OpenHands condensation and conversation surfaces where memory or history affects context. | Memory promotion can convert transient or disputed context into durable recallable state without contradiction metadata. | Can public artifacts distinguish stored memory, recalled memory, summary-derived memory, and contradicted memory? | unresolved |
| summary contradiction survivability | Preserving contradictions, caveats, omitted ranges, and source refs inside or alongside compressed summaries. | Summary quality determines whether later context can reconstruct ambiguity rather than only a simplified narrative. | Compaction summaries, condenser outputs, omitted-event refs, successor transcripts, and condensed LLM-ready views. | Summaries can resolve conflict implicitly, omit low-confidence branches, or lose links to original events. | What metadata is required for a summary to be contradiction-preserving rather than contradiction-flattening? | hypothesis |
| operational truth survivability | Preserving enough evidence lineage to distinguish operationally supported facts from simplified, inferred, stale, or contradicted narrative. | Public operational proofs depend on knowing whether replayed truth is evidence-backed, compressed, inferred, or unresolved. | Combined transcript, event, memory, tool/action, environment, replay, summary, and provenance artifacts. | Operational truth can degrade when compression, replay, and memory promotion preserve utility but lose evidence boundaries. | Which minimum lineage bundle is enough to treat a reconstructed claim as operationally supported? | unresolved |
