# Replay Dimension Model

The dimensions below define what an operational cognition replay audit may need to reconstruct. The current statuses are preliminary and subject to evidence validation.

| Dimension | Definition | Why It Matters | Evidence Needed | Public Docs Appear To Address It | Current Status |
| --- | --- | --- | --- | --- | --- |
| visible transcript replay | Replaying visible user, assistant, and session transcript messages in order. | Transcript replay can show what was said, but not necessarily why an action path was selected. | Session transcript format, message ordering rules, redaction rules, retention boundaries. | OpenClaw trajectory docs and OpenHands conversation docs appear to address portions of this. | observed |
| tool/action sequence replay | Reconstructing ordered tool calls, action events, and observations. | Tool/action order is needed to inspect operational pathways and handoffs. | Tool call logs, action IDs, observation IDs, timestamps, event ordering guarantees. | OpenClaw trajectory docs and OpenHands event/conversation docs appear to address portions of this. | observed |
| tool result replay | Reconstructing the returned content, errors, truncation, and provenance for tool results. | Tool results can drive later choices; omitted or truncated results can change audit interpretation. | Tool result bodies or hashes, truncation markers, redaction policy, source URLs, error metadata. | Public docs indicate some tool result capture, but exact adequacy remains unresolved. | unresolved |
| environment/workspace state replay | Reconstructing filesystem, workspace, sandbox, container, and external state references. | Agent behavior can depend on files, installed tools, ports, credentials presence, and sandbox state. | Environment snapshots, workspace hashes, container image IDs, filesystem diffs, external dependency references. | OpenHands agent-server docs appear relevant; OpenClaw trajectory docs mention metadata but completeness is unresolved. | inferred |
| browser/UI state replay | Reconstructing browser DOM, navigation, scrolling, and UI interactions. | Browser replay may show visible behavior but may not reproduce network, auth, or hidden application state. | Browser recordings, rrweb events, page URLs, timing, network references, auth/session boundaries. | OpenHands browser recording docs and rrweb docs appear to address visible UI replay. | observed |
| memory state replay | Reconstructing durable memory, short-term notes, recall indexes, and memory search inputs used during the run. | Memory can alter decisions across sessions and after compaction. | Memory file snapshots, memory index metadata, recall queries, recall results, promotion/flush markers. | OpenClaw memory docs appear relevant; OpenHands condenser docs address context compression, not full memory replay. | unresolved |
| compressed context replay | Reconstructing summaries, forgotten event IDs, compaction boundaries, and model-visible context after compression. | Long-horizon replay may diverge if the audit sees full transcript but the model saw a summary. | Summary text, forgotten IDs, context window boundaries, kept head/tail rules, compaction timestamps. | OpenClaw compaction docs and OpenHands condenser docs appear to address portions of this. | observed |
| configuration/runtime replay | Reconstructing model, provider, tool policy, sandbox, plugin, skill, and runtime configuration. | Runtime configuration changes can change available actions, approvals, and model behavior. | Config snapshots, model IDs, tool schemas, plugin/skill lists, sandbox mode, runtime image or process metadata. | OpenClaw trajectory docs and OpenHands agent-server/conversation docs appear relevant. | inferred |
| recovery loop replay | Reconstructing retries, fallback models, aborts, resumptions, lost tasks, and cleanup behavior. | Failure pathways often depend on recovery behavior rather than only the first error. | Retry traces, fallback events, task state transitions, cleanup status, restart markers, timeout metadata. | Public docs mention some runtime and task status surfaces, but current Proof 002 evidence does not yet establish adequacy. | unresolved |
| causal decision lineage replay | Reconstructing why a decision was made from visible context, memory, tool results, configuration, and model-visible prompt state. | Operational cognition audits need causal explanation boundaries, not only visible playback. | Model-visible context hashes, prompt-building metadata, tool schemas, memory recalls, environment hashes, event lineage. | Public evidence currently indicates partial lineage surfaces, but causal adequacy remains unresolved. | unresolved |
