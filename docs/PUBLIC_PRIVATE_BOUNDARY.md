# Public/Private Boundary

Operational Cognition publishes sanitized public proof artifacts. It does not publish raw cognition dumps, private substrate state, or implementation internals.

## Public

The public repository may contain:

- problem framing
- evidence references
- failure taxonomies
- public lineage summaries
- mitigation doctrine
- proof manifests

## Private

The public repository must not contain:

- Belief Ledger implementation
- Inside Voice implementation
- MCP internals beyond public endpoint contract references
- harmonic traceback details
- private scoring
- hidden reasoning chains
- substrate state

## Boundary Rule

Public artifacts are sanitized outputs, not raw cognition dumps.

Private systems may assist planning, review, or continuity governance, but public proof packs must stand on public evidence, public summaries, explicit uncertainty, and deterministic manifests. A reader must not need access to private substrate internals to understand the public claim boundary or replay the public investigation trail.

## Ignored Internal Artifacts

The `internal/` directory is intentionally ignored. Public proof packs must not depend on private raw consultation artifacts.
