# Operational Cognition

## Purpose

Operational Cognition is a public proof surface for continuity-governed AI engineering investigations.

This repository publishes disciplined, replayable artifacts for examining long-horizon AI engineering behavior without exposing private substrate implementation, hidden reasoning chains, or internal cognition-routing machinery.

## Why this exists

Modern AI engineering systems increasingly struggle with long-horizon continuity, replayability, architectural drift, agent failure reconstruction, and provenance loss. These problems do not require speculative claims about intelligence or consciousness. They require operational evidence, deterministic lineage, bounded claims, and a clear separation between public proof artifacts and private infrastructure.

Operational Cognition exists to make that public layer explicit.

See: [docs/OPERATIONAL_COGNITION_THESIS.md](docs/OPERATIONAL_COGNITION_THESIS.md)

## What this repo publishes

- public proof packs
- failure taxonomies
- evidence manifests
- replayability plans
- mitigation doctrine
- public lineage summaries

## What this repo does not publish

- private substrate implementation
- hidden chain-of-thought
- Belief Ledger internals
- Inside Voice internals
- proprietary cognition routing
